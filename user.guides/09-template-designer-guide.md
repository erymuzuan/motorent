# Document Template Designer Guide

The Document Template Designer is a visual, drag-and-drop tool that allows Organization Administrators to customize the documents generated by JaleOS, such as Rental Agreements, Booking Confirmations, and Receipts.

## Overview

JaleOS uses a section-based layout system. Each template consists of multiple sections (Header, Body, Footer) containing draggable "blocks" like text, tables, and images. These blocks can be bound to dynamic data from your rentals and bookings.

## Getting Started

1. Navigate to **Settings > Document Templates**.
2. Click **Create New Template** or click on an existing template to edit it.
3. Select the **Template Type**:
   - **Booking Confirmation**: Sent to customers when they reserve a vehicle.
   - **Rental Agreement**: The legal contract signed at check-in.
   - **Receipt**: The financial proof of payment.

## Using the Designer

The designer interface is split into three main areas:
1. **The Toolbox (Left)**: Contains draggable blocks.
2. **The Canvas (Center)**: Where you arrange your blocks.
3. **Property Editor (Right)**: Configure the settings for the selected block.

### Draggable Blocks

| Block Type | Description |
|------------|-------------|
| **Text** | Static text or dynamic data tokens (e.g., `{{CustomerName}}`). |
| **Image** | Your logo or other branding images. |
| **Table** | Used for itemized lists like vehicle details or payment breakdowns. |
| **Separator** | Horizontal lines to divide sections. |
| **Header/Footer** | Pre-configured blocks for standard document headers and footers. |

### Data Binding (Tokens)

You can insert dynamic data into Text blocks using double curly braces.

**Common Tokens:**
- `{{OrganizationName}}` - Your business name.
- `{{ShopName}}` - The specific shop location.
- `{{CustomerName}}` - Full name of the renter.
- `{{VehicleName}}` - Make and model of the bike.
- `{{StartDate}}` / `{{EndDate}}` - Rental period.
- `{{TotalAmount}}` - Total cost of the rental.

## Best Practices

1. **Test with Preview**: Always use the "Preview" button to see how your template looks with sample data before making it live.
2. **Set as Default**: Once you are happy with a template, click **Set as Default** so it is automatically used for all new rentals/bookings.
3. **Branding**: Upload a high-resolution logo (PNG or JPG) for the best print quality.
4. **Legal Compliance**: Ensure your Rental Agreement includes all necessary terms and conditions required by local laws.

## Template Status

- **Draft**: Work in progress, not visible to staff.
- **Approved**: Ready for use, can be manually selected when printing.
- **Default**: The primary template used automatically by the system.

## Troubleshooting

### Tokens not showing data
Ensure you are using the correct token names (case-sensitive) and that the token is valid for the template type (e.g., `{{BookingID}}` only works on Booking templates).

### Layout looks broken on mobile
The designer is optimized for desktop use. For the best experience, design your templates on a large screen.
