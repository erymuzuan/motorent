# Specification: MotoRent Documentation Portal (/learn)

## Overview
This track involves the creation of a centralized, public-facing documentation portal at `/learn`. The portal will serve as the primary resource for tourists, staff, mechanics, and administrators. It includes a foundational task to audit the current codebase and update existing markdown guides to ensure they reflect the latest features and logic.

## Functional Requirements

### 1. Documentation Audit & Update
- **Source Audit:** Review current `src/` logic and UI to identify missing or outdated documentation in the `user.guides/` folder.
- **Content Update:** Refine all `.md` files to ensure accuracy regarding current features (e.g., OCR, document templates, rental workflows).
- **Categorization:** Organize guides into clear roles: Tourist, Staff, Manager, Mechanic, and Super Admin.

### 2. Documentation Portal UI (/learn)
- **Portal Layout:** Implement a classic documentation style with a fixed sidebar for navigation and a main content area.
- **Runtime Rendering:** Fetch `.md` files from `wwwroot/user.guides` and render them to HTML at runtime using a library like Markdig.
- **Responsive Design:** Ensure the portal is fully functional and readable on mobile devices.

### 3. Gemini AI Integration
- **Natural Language Search:** A search bar that allows users to ask questions in plain English or Thai (e.g., "How do I process a deposit refund?").
- **Gemini Chat Interface:** A dedicated chat component that uses the documentation as a knowledge base (RAG) to provide direct answers.
- **Link Citation:** Gemini should provide links to the specific documentation sections it used to answer a query.

## Technical Implementation
- **Rendering:** Client-side (Blazor) rendering using Markdig, fetching source files via HTTP from `wwwroot`.
- **Content Source:** Markdown files deployed to `wwwroot/user.guides`.
- **Search Logic:** Gemini Flash API for natural language processing and context-aware search.
- **Navigation:** Dynamic sidebar generated by scanning the `user.guides` directory or using a manifest file.

## Acceptance Criteria
- [ ] All existing user guides are reviewed and updated based on current source code.
- [ ] The `/learn` page is accessible publicly and loads quickly.
- [ ] Navigation sidebar correctly links to all categories and guides.
- [ ] Search bar successfully returns relevant documentation sections for natural language queries.
- [ ] Gemini chat provides accurate answers based *only* on the provided MotoRent guides.
- [ ] Documentation is fully readable and navigable on mobile screens.

## Out of Scope
- Converting the documentation portal into a full CMS (content management system).
- Video tutorials (handled as external links if they exist).
