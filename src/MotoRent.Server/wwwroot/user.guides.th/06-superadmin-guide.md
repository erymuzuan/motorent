# คู่มือเริ่มต้นใช้งาน MotoRent - Super Admin

ยินดีต้อนรับสู่ MotoRent! คู่มือนี้ครอบคลุมบทบาท Super Admin (ผู้ดูแลแพลตฟอร์ม)

![Impersonate User](images/01-impersonate.png)

## บทบาทของคุณ

ในฐานะ Super Admin คุณจัดการแพลตฟอร์ม MotoRent ทั้งหมด:
- จัดการองค์กร (Tenant)
- จัดการผู้ใช้ระบบ
- สวมรอยผู้ใช้สำหรับการสนับสนุน
- จัดการคำเชิญลงทะเบียน
- ดู System Logs
- กำหนดค่าการตั้งค่าทั่วไป

> **หมายเหตุ:** Super Admin เป็นบทบาทระดับแพลตฟอร์ม หากไม่มีการสวมรอย คุณจะไม่เห็นเมนู Tenant (การเช่า ยานพาหนะ ฯลฯ) ใช้ฟีเจอร์สวมรอยเพื่อเข้าถึงฟีเจอร์เฉพาะของ Tenant

## การเข้าถึงฟีเจอร์ Super Admin

### การนำทาง

หน้า Super Admin เข้าถึงผ่านเมนูผู้ใช้แบบดรอปดาวน์:
1. คลิกไอคอนโปรไฟล์ของคุณที่มุมขวาบน
2. เลือก "Super Admin" หรือไปยังหน้า `/super-admin/*` โดยตรง

### หน้า Super Admin

| หน้า | URL | วัตถุประสงค์ |
|------|-----|---------|
| องค์กร | `/super-admin/organizations` | จัดการ Tenant |
| ผู้ใช้ | `/super-admin/users` | จัดการผู้ใช้ระบบ |
| สวมรอย | `/super-admin/impersonate` | สนับสนุนการสวมรอยผู้ใช้ |
| คำเชิญ | `/super-admin/invites` | รหัสคำเชิญลงทะเบียน |
| Logs | `/super-admin/logs` | บันทึกข้อผิดพลาดระบบ |
| การตั้งค่า | `/super-admin/settings` | การกำหนดค่าทั่วไป |

## การจัดการองค์กร

### การดูองค์กร

ไปที่ **Super Admin > องค์กร** เพื่อดู Tenant ทั้งหมด:
- ชื่อองค์กร
- หมายเลขบัญชี (ตัวระบุเฉพาะ)
- ข้อมูลติดต่อ
- สถานะ (ใช้งาน/ไม่ใช้งาน)
- วันที่สร้าง

### การสร้างองค์กร

1. คลิก "เพิ่มองค์กร"
2. กรอกรายละเอียดที่จำเป็น:
   - **ชื่อองค์กร** - ชื่อธุรกิจ
   - **หมายเลขบัญชี** - ตัวระบุเฉพาะ (สร้างอัตโนมัติหรือกำหนดเอง)
   - **อีเมลติดต่อ** - ผู้ติดต่อหลัก
   - **โทรศัพท์ติดต่อ** - โทรศัพท์ธุรกิจ
3. กำหนดค่าการตั้งค่าเริ่มต้น
4. คลิก "สร้าง"

### การแก้ไของค์กร

1. ค้นหาองค์กรในรายการ
2. คลิก "แก้ไข" หรือชื่อองค์กร
3. อัปเดตรายละเอียดตามต้องการ
4. บันทึกการเปลี่ยนแปลง

### สถานะองค์กร

- **ใช้งาน** - การดำเนินงานปกติ
- **ไม่ใช้งาน** - ถูกระงับ (ผู้ใช้ไม่สามารถล็อกอินได้)

## การจัดการผู้ใช้

### การดูผู้ใช้

ไปที่ **Super Admin > ผู้ใช้** เพื่อดูผู้ใช้ระบบทั้งหมด:
- ชื่อผู้ใช้และอีเมล
- ผู้ให้บริการยืนยันตัวตน (Google, Microsoft)
- องค์กรที่เกี่ยวข้อง
- บทบาทภายในแต่ละองค์กร
- วันที่ล็อกอินล่าสุด

### บัญชีผู้ใช้

ผู้ใช้แต่ละคนสามารถเป็นสมาชิกขององค์กรหลายแห่งด้วยบทบาทที่แตกต่างกัน:

| บทบาท | คำอธิบาย |
|------|-------------|
| OrgAdmin | สิทธิ์เข้าถึงเต็มรูปแบบในองค์กร |
| ShopManager | จัดการการดำเนินงานของร้าน |
| Staff | จัดการการเช่าประจำวัน |
| Mechanic | การบำรุงรักษายานพาหนะ |

### การเพิ่มผู้ใช้ในองค์กร

1. ค้นหาผู้ใช้
2. คลิก "จัดการสิทธิ์"
3. เลือกองค์กร
4. กำหนดบทบาท
5. บันทึกการเปลี่ยนแปลง

## การสวมรอยผู้ใช้

ฟีเจอร์สวมรอยช่วยให้คุณลงชื่อเข้าใช้ในฐานะผู้ใช้อื่นสำหรับวัตถุประสงค์การสนับสนุน

### วิธีการสวมรอย

1. ไปที่ **Super Admin > สวมรอย**
2. ค้นหาผู้ใช้ด้วย:
   - ชื่อผู้ใช้
   - อีเมล
   - ชื่อ
3. คลิกป้ายองค์กรเพื่อสวมรอย
4. ตอนนี้คุณลงชื่อเข้าใช้ในฐานะผู้ใช้นั้น

### ระหว่างการสวมรอย

- คุณเห็นระบบเหมือนกับที่ผู้ใช้นั้นเห็น
- การดำเนินการของคุณถูกบันทึกในฐานะผู้ใช้ที่ถูกสวมรอย
- แบนเนอร์แสดงว่าคุณกำลังสวมรอย
- คลิก "หยุดสวมรอย" เพื่อกลับไปยังบัญชีของคุณ

### เมื่อใดควรใช้การสวมรอย

- **การแก้ไขปัญหา** - ดูสิ่งที่ผู้ใช้เห็น
- **การสนับสนุน** - ช่วยผู้ใช้กับปัญหาเฉพาะ
- **การทดสอบ** - ยืนยันว่าฟีเจอร์ทำงานถูกต้อง
- **การฝึกอบรม** - สาธิตฟีเจอร์

### แนวทางปฏิบัติที่ดี

1. สวมรอยเฉพาะเมื่อจำเป็น
2. อย่าทำการเปลี่ยนแปลงเว้นแต่ได้รับการร้องขอ
3. สิ้นสุดการสวมรอยเมื่อเสร็จสิ้น
4. บันทึกการโต้ตอบการสนับสนุน

## คำเชิญลงทะเบียน

รหัสเชิญช่วยให้องค์กรใหม่ลงทะเบียนบนแพลตฟอร์ม

### การสร้างคำเชิญ

1. ไปที่ **Super Admin > คำเชิญ**
2. คลิก "สร้างคำเชิญ"
3. กำหนดค่า:
   - **รหัส** - กำหนดเองหรือสร้างอัตโนมัติ
   - **การใช้งาน** - จำนวนการใช้งานสูงสุด
   - **หมดอายุ** - วันหมดอายุ
   - **เทมเพลตองค์กร** - การตั้งค่าที่กำหนดไว้ล่วงหน้า
4. คลิก "สร้าง"

### การจัดการคำเชิญ

- ดูคำเชิญที่ใช้งานอยู่และหมดอายุ
- ดูสถิติการใช้งาน
- เพิกถอนคำเชิญที่ไม่ได้ใช้
- ขยายวันหมดอายุ

### ขั้นตอนการทำงานของคำเชิญ

1. สร้างรหัสเชิญ
2. แชร์กับลูกค้าใหม่
3. ลูกค้าลงทะเบียนโดยใช้รหัส
4. องค์กรถูกสร้างขึ้น
5. ลูกค้าตั้งค่าบัญชีของตน

## System Logs

ดูและวิเคราะห์เหตุการณ์และข้อผิดพลาดของระบบ

### การดู Logs

ไปที่ **Super Admin > Logs** เพื่อดู:
- ข้อความแสดงข้อผิดพลาด
- Stack traces
- ผู้ใช้/องค์กรที่ได้รับผลกระทบ
- Timestamps

### การกรอง Logs

กรองตาม:
- **ระดับ** - Error, Warning, Info
- **ช่วงวันที่** - ช่วงเวลาที่กำหนดเอง
- **องค์กร** - Tenant เฉพาะ
- **ผู้ใช้** - ผู้ใช้เฉพาะ

### ประเภท Log ทั่วไป

| ประเภท | คำอธิบาย |
|------|-------------|
| Error | ข้อผิดพลาดระบบที่ต้องการความสนใจ |
| Warning | ปัญหาที่อาจเกิดขึ้น |
| Info | เหตุการณ์ระบบทั่วไป |
| Audit | การดำเนินการของผู้ใช้ (ล็อกอิน การเปลี่ยนแปลง) |

## การตั้งค่าระบบ

กำหนดค่าการตั้งค่าแพลตฟอร์มทั่วไป

### การตั้งค่าทั่วไป

- ชื่อแพลตฟอร์มและการสร้างแบรนด์
- ภาษาเริ่มต้น
- รูปแบบวันที่/เวลา
- การตั้งค่าสกุลเงิน

### การตั้งค่ายืนยันตัวตน

- ผู้ให้บริการ OAuth (Google, Microsoft)
- เวลาหมดอายุเซสชัน
- นโยบายรหัสผ่าน (ถ้ามี)

### การตั้งค่าอีเมล

- การกำหนดค่า SMTP
- เทมเพลตอีเมล
- การตั้งค่าการแจ้งเตือน

### Feature Flags

เปิด/ปิดฟีเจอร์แพลตฟอร์ม:
- พอร์ทัลนักท่องเที่ยว
- การสนับสนุนหลายสถานที่
- การรายงานขั้นสูง
- การเข้าถึง API

## การบริหารประจำวัน

### รายการตรวจสอบตอนเช้า

- [ ] ตรวจสอบ System Logs สำหรับข้อผิดพลาด
- [ ] ตรวจสอบคำเชิญลงทะเบียนที่รอดำเนินการ
- [ ] ยืนยันว่าองค์กรทั้งหมดใช้งานอยู่
- [ ] ตรวจสอบ Support Tickets ใดๆ

### งานรายสัปดาห์

- [ ] ตรวจสอบสิทธิ์การเข้าถึงของผู้ใช้
- [ ] ตรวจสอบการใช้งานขององค์กร
- [ ] ตรวจสอบประสิทธิภาพระบบ
- [ ] อัปเดตเอกสาร

### งานรายเดือน

- [ ] สร้างรายงานแพลตฟอร์ม
- [ ] ตรวจสอบการตั้งค่าความปลอดภัย
- [ ] วางแผนการอัปเดตระบบ
- [ ] สำรองการกำหนดค่า

## ข้อควรพิจารณาด้านความปลอดภัย

### การควบคุมการเข้าถึง

1. จำกัดการเข้าถึง Super Admin ให้กับบุคลากรที่จำเป็น
2. ใช้การยืนยันตัวตนที่แข็งแกร่ง (OAuth)
3. ตรวจสอบ Access Logs อย่างสม่ำเสมอ
4. ลบบัญชีที่ไม่ได้ใช้

### การปกป้องข้อมูล

1. อย่าแชร์ข้อมูลรับรองผู้ใช้
2. ใช้การสวมรอยสำหรับการสนับสนุน (ไม่ใช่การล็อกอินโดยตรง)
3. บันทึกการดำเนินการบริหารทั้งหมด
4. ปฏิบัติตามนโยบายการเก็บรักษาข้อมูล

### การตอบสนองต่อเหตุการณ์

หากคุณตรวจพบกิจกรรมที่น่าสงสัย:
1. บันทึกปัญหา
2. ปิดใช้งานบัญชีที่ได้รับผลกระทบชั่วคราว
3. ตรวจสอบ Logs สำหรับขอบเขต
4. แจ้งฝ่ายที่ได้รับผลกระทบ
5. ดำเนินมาตรการแก้ไข

## การแก้ไขปัญหา

### ปัญหาทั่วไป

**ผู้ใช้ไม่สามารถล็อกอิน:**
1. ตรวจสอบสถานะผู้ใช้ (ใช้งาน)
2. ยืนยันสถานะองค์กร
3. ตรวจสอบผู้ให้บริการยืนยันตัวตน
4. ตรวจสอบ Log Entries ล่าสุด

**องค์กรไม่ปรากฏ:**
1. ยืนยันว่าองค์กรมีอยู่
2. ตรวจสอบสถานะ (ใช้งาน/ไม่ใช้งาน)
3. ยืนยันว่าผู้ใช้มีสิทธิ์เข้าถึง

**การสวมรอยไม่ทำงาน:**
1. ยืนยันบทบาท Super Admin
2. ตรวจสอบว่าผู้ใช้เป้าหมายมีอยู่
3. ตรวจสอบให้แน่ใจว่าองค์กรใช้งานอยู่

## การขอความช่วยเหลือ

- **เอกสารทางเทคนิค** - ตรวจสอบเอกสารภายใน
- **ทีมพัฒนา** - ติดต่อสำหรับปัญหาระบบ
- **ทีมรักษาความปลอดภัย** - รายงานข้อกังวลด้านความปลอดภัย
